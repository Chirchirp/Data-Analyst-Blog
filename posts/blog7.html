One Platform to Rule All Your Data — Microsoft Fabric

Microsoft Fabric promised to end the fragmentation wars in enterprise analytics. After two years of broad availability, the question isn't whether it's ambitious — it's whether the ambition holds up when real data hits real pipelines.

Pharaoh Chirchir • March 2026 • 10 min read
Microsoft Fabric Data Engineering OneLake Medallion Power BI

If you've spent time in the Microsoft data ecosystem, you know the old drill: Azure Data Factory for ingestion, Synapse Analytics for warehousing, Power BI for reporting, Azure Machine Learning for models — each with its own billing, access controls, monitoring, and mental model. Powerful, yes. Coherent, decidedly not.

Microsoft Fabric is the answer to that fragmentation. Launched in 2023 and now deeply embedded in enterprise data stacks worldwide, it bundles the full analytics lifecycle under one roof — one workspace, one governance model, one storage layer called OneLake. I've spent the past year implementing it across several organisations. Here's what actually works, what doesn't, and how the pieces fit together.

"OneLake is what makes Fabric feel like an integrated product rather than a rebranded bundle. Data written once is data accessible everywhere — no movement, no duplication, no drift."

What Fabric Actually Is — Beyond the Marketing

Fabric is best understood as six interconnected experiences sharing a single storage foundation. The mistake most teams make is treating it like a new product to learn from scratch. It's more accurate to think of it as the layer that finally connects the Microsoft tools you were already buying separately.

Six core experiences
01

Data Factory — Ingestion & Orchestration

150+ connectors, visual ETL/ELT pipeline builder, Copy Activity, Dataflows Gen2, scheduling and retry logic. The connectors cover on-prem databases, SaaS platforms, cloud storage and real-time event sources. This is your primary tool for moving data into OneLake.

02

Data Engineering — Spark Lakehouse

Spark notebooks (Python, Scala, R, SQL) running against Delta Lake tables in OneLake. The Lakehouse is the primary workspace for Bronze-to-Silver transformations. Incremental loads, schema enforcement, complex joins — all handled here without leaving the Fabric workspace.

03

Data Warehouse — T-SQL Governed Queries

A fully managed SQL warehouse that reads Delta tables in OneLake directly. No data movement, no import step. Analysts who live in SQL can query the same Gold-layer data that Power BI uses. Row-level security, column-level encryption, views — all standard SQL primitives.

04

Data Science — ML & Experimentation

MLflow-integrated notebooks for model training, tracking and deployment. The Silver layer tables feed the feature store directly. Models trained here can score into new columns that the Gold layer and Power BI then consume — closing the loop from raw data to decision.

05

Real-Time Intelligence — KQL & Eventstream

KQL databases for sub-second analytics on high-velocity data. Eventstream captures continuous feeds from Event Hubs, Kafka, and IoT Hub with built-in routing and transformations. The part of Fabric with the steepest learning curve — but the only viable option for genuine real-time dashboarding.

06

Power BI — Reporting & Semantic Models

Now a first-class Fabric citizen rather than an adjacent tool. DirectLake mode reads Delta files in OneLake directly — no import, no scheduled refresh lag. Semantic models define DAX measures and row-level security once, reused across all reports. Copilot writes DAX and generates narratives inline.

OneLake: Why the Storage Model Changes Everything

The fragmentation problem in enterprise data isn't primarily about tools — it's about data copies. Every time data moves from one system to another, you create a synchronisation problem, a latency window, and a governance gap. Most data platforms solve the tool problem while making the copy problem worse.

OneLake inverts this. It's a single, tenant-wide data lake built on Azure Data Lake Storage Gen2, using the open Delta Parquet format. Every Fabric experience — Spark notebooks, SQL Warehouse, Power BI, Real-Time Intelligence — reads and writes to the same physical files. Your Spark notebook writes a Delta table. Your SQL Warehouse queries it directly. Power BI reads it in DirectLake mode without an import step.

The key insight

OneLake Shortcuts extend this further — virtual links to external storage (S3, ADLS Gen2, GCS) that allow Fabric to query data in place without ever copying it. Multi-cloud data estates become queryable from a single namespace without consolidation costs.

Practical note on DirectLake

DirectLake mode gives Power BI report-speed performance against OneLake Delta tables — but it falls back to DirectQuery automatically when queries exceed what DirectLake can handle (e.g., complex DAX against very large tables). Monitor the fallback rate in your semantic model metrics; consistent fallback is a signal to review your Gold-layer table design or add aggregations.

Where Fabric Genuinely Works — and Where It Doesn't

Fabric has earned real enthusiasm in the data community. That enthusiasm is partly warranted and partly premature. Here's a breakdown by capability, based on production experience:

What works well

  • Unified billing and capacity model (F-SKUs)
  • Single identity & governance via Microsoft Purview
  • No data movement between engines
  • Git integration for CI/CD pipelines
  • Copilot across all experiences
  • OneLake Shortcuts for multi-cloud access
  • Strong Microsoft 365 data integration

Where friction remains

  • Capacity pricing opaque at scale
  • SQL Warehouse lags Synapse on features
  • Spark cold-start times on small clusters
  • Multi-tenant governance still maturing
  • KQL / Real-Time Intelligence steep curve
  • Third-party connector depth varies

Who Should Be Looking at Fabric Right Now

The answer depends almost entirely on where you're starting from, not where you want to end up.

Strong fit: existing Microsoft shops

If you're already running Azure, Power BI, Microsoft 365, and Dynamics — Fabric is an almost obvious evolution. Your Teams data, SharePoint content, and Dynamics CRM flow into OneLake with minimal plumbing. The governance story is already half-built through Entra ID and Purview. The incremental value is high, the switching cost is low.

Worth evaluating: mid-market consolidation plays

If your data stack is fragmented — separate ETL tool, separate warehouse, separate BI layer — Fabric's unified workspace and single billing model can cut real operational overhead. You trade some best-of-breed capability for coherence. For many mid-market organisations, that's the right trade.

Harder case: mature Databricks or Snowflake shops

Fabric doesn't yet match Databricks' Unity Catalog maturity or Snowflake's data sharing ecosystem. The pace of development is fast, but "catching up" isn't "caught up." Migration costs need to be weighed against realistic capability gaps, not the roadmap.

Key takeaways

  • Fabric's real innovation is OneLake — a shared storage foundation that eliminates data copies between pipeline stages. Everything else is built on top of this.
  • The Medallion architecture (Bronze → Silver → Gold) gives Fabric a clear data quality progression that works well with the Lakehouse paradigm.
  • Six distinct ingestion patterns cover almost every source type: Pipelines, Eventstream, Spark, Mirroring, Shortcuts, and Dataflows Gen2.
  • DirectLake mode is a genuine step change for Power BI performance — but monitor for DirectQuery fallback on complex semantic models.
  • The SQL Warehouse, Real-Time Intelligence, and multi-tenant governance layers are improving fast but not yet at full parity with dedicated alternatives.
  • Microsoft ecosystem shops have the strongest value proposition. Greenfield or Databricks/Snowflake migrations need a careful capability-gap assessment first.
TL;DR

Microsoft Fabric is the most coherent attempt yet to collapse the enterprise data platform stack into something a single team can actually own. The OneLake foundation and Medallion architecture give it a genuine structural advantage over bolt-together alternatives. The gaps — SQL Warehouse feature parity, Real-Time Intelligence complexity, multi-tenant governance — are real but narrowing. If you're in the Microsoft ecosystem and managing a fragmented data stack, Fabric deserves more than a demo. It deserves a serious pilot.
